📋 Project Overview

SmartGate is a web-based university campus vehicle access control system that uses UHF RFID tags to manage the entry and exit of registered vehicles belonging to students, faculty, and non-teaching staff. It also supports manual visitor management for non-registered guests. The system is designed to replace manual logbooks at university gates with a fast, verifiable, and digitally tracked process.

Purpose: Ensure only authorized, registered vehicles enter the university campus, log all gate activity, and provide administrators with real-time oversight and historical reports.

Online Pre-Registration

At /online-registration, any member of the public (student, faculty, or staff) can submit a vehicle registration application without logging in.

  • Personal info: name, role, university/faculty/employee ID, department, contact number, email
  • Vehicle info: type (car, SUV, van, motorcycle), make/brand, plate number
  • Document uploads: Certificate of Registration (CR), Official Receipt (OR), Driver's License, COR, Student/Employee ID, Business Permit (for staff)
  • Each document is validated in real-time using the DocumentValidationService upon upload
  • On successful submission, all Admin and Office users are automatically notified by email
  • The registration is saved with a pending status awaiting office review

RFID Tag Assignment (Hardware Scanner Integration)

When assigning an RFID tag during registration:

  • Click "Scan Tag" — the browser automatically connects to bridge_service.py via WebSocket (ws://127.0.0.1:8080)
  • Double Scan Verification ensures the correct tag is captured:
ScanStateUI ColorMessage
1st Scan Pending Yellow input border "Step 1/2 — Scan again to confirm"
2nd Scan (match) Verified Green input border, field filled "Tag Verified! UID: xxxx"
2nd Scan (mismatch) Mismatch Red input border, auto-reset "Tag Mismatch — try again"
No 2nd scan in 10s Timeout State reset Bridge auto-clears the pending tag
  • Manual entry mode is available as a fallback — type and confirm a tag ID manually
  • Duplicate tag check runs before final submission to prevent assigning an already-used tag

🚧 Guard Portal

Dashboard

RFID Gate Scanning

  • Guard presents a vehicle's RFID tag to the UHF reader
  • System calls lookupTag — retrieves the registered owner and current status (in/out)
  • System calls logVehicle — logs as entry or exit (smart toggle: if the last log was entry, the next is automatically exit)
  • Unregistered tags are logged with an "Unregistered" label — not blocked, just flagged
  • All scan actions are written to the audit log

Visitor Entry (Manual)

  • Guard fills out a quick form: visitor name, plate number, vehicle type, purpose, destination
  • Record saved with status: inside and a time_in timestamp

Visitor Exit

  • Shows all visitors currently with status: inside
  • Guard selects a visitor to mark as departed: status updates to left, time_out is recorded

🔌 Hardware Bridge — bridge_service.py

A standalone Python script that runs on the same PC as the UHF RFID USB reader. It acts as the translator between the physical hardware and the Laravel web application.

ComponentDetail
Serial PortCOM5, Baud Rate: 9600
Reader TypeUHF RFID (ISO 18000-6C)
WebSocket Serverws://0.0.0.0:8080
Debounce Window2 seconds per unique tag (prevents duplicate reads)
Verification Timeout10 seconds (auto-clears unconfirmed pending tag)
Threading ModelSerial port runs in a daemon thread; WebSocket runs on asyncio event loop
Queueasyncio.Queue for non-blocking, zero-delay push from serial thread to WebSocket handler

Double Scan State Machine (Backend Logic)

  • 1st scan → tag stored as PENDING_TAG, sends status: "pending" to frontend
  • 2nd scan within 10s, same tag → sends status: "verified", clears state
  • 2nd scan within 10s, different tag → sends status: "mismatch", clears state
  • No 2nd scan within 10s → state auto-resets on the next scan attempt
